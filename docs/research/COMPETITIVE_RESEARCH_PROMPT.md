You are conducting competitive research to build a comprehensive feature backlog for **niche-selection-app**, a market research and niche analysis tool. Your goal is to identify 150-300+ features by analyzing competitors and industry best practices.

- Entrepreneurs identify profitable market niches

**Current Tech Stack (Already Modern):**
- Frontend: React 19, Vite 6, TypeScript
- Backend: Express, Prisma
- Data: Google Trends API, YouTube API
- UI: Tailwind CSS (upgrading to v4)

## Output Format

Provide your research in this format:

### Feature Backlog Structure

```markdown
## Category X: [Category Name] (X features)

### P0 - Critical (MVP Required)
| ID | Feature | Description | Effort | Competitors |
|----|---------|-------------|--------|-------------|
| FX.X.X | Feature Name | What it does | Low/Med/High | JungleScout, Ahrefs |

### P1 - High Priority (Competitive Parity)
[Same table format]

### P2 - Medium Priority (Differentiation)
[Same table format]

### P3 - Future (Nice to Have)
[Same table format]
```

### Priority Definitions
- **P0 Critical**: Core features needed for MVP launch
- **P1 High**: Features that major competitors all have
- **P2 Medium**: Features that differentiate from competitors
- **P3 Future**: Advanced features for long-term roadmap

### Effort Definitions
- **Low**: 1-2 days implementation
- **Medium**: 3-5 days implementation
- **High**: 1-2 weeks implementation
